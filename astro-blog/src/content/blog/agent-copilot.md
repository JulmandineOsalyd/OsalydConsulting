Depuis quelques temps, on entend le mot « Agent » associé à l'IA à toutes les sauces. Honnêtement, même moi je n'étais plus totalement sûre de ce qu'était un agent dans le contexte Microsoft 365. Alors, remettons un peu d'ordre dans tout ça.

## Définition simple d'un agent

Un agent IA est un programme qui peut prendre des décisions et effectuer des actions de manière autonome pour atteindre un objectif défini. Contrairement à un simple chatbot qui répond à des questions, un agent peut :

- Planifier une série d'étapes
- Utiliser des outils (recherche web, exécution de code, appels API)
- S'adapter en fonction des résultats obtenus

## Les types d'agents dans Microsoft 365

### Agents déclaratifs

Ce sont des agents basés uniquement sur des instructions textuelles (le "prompt"). Ils sont simples à créer mais limités dans leurs capacités. Parfaits pour commencer.

### Agents personnalisés (Custom Engine Agents)

Ces agents utilisent votre propre modèle IA et votre logique métier. Plus complexes mais beaucoup plus puissants.

### Agents automatisés

Ces agents sont déclenchés par des événements (un email reçu, une ligne ajoutée dans Excel, etc.) et exécutent des tâches sans intervention humaine.

## Où créer des agents dans Microsoft 365 ?

- **Copilot Chat** : pour les agents déclaratifs simples
- **Copilot Studio** : pour des agents plus sophistiqués avec des connecteurs et des flux
- **Power Automate** : pour les agents automatisés basés sur des déclencheurs

## La différence avec Copilot "classique"

Copilot est un assistant IA réactif : vous lui posez une question, il répond. Un agent est proactif : il peut agir de lui-même pour accomplir une tâche complexe, en plusieurs étapes.
